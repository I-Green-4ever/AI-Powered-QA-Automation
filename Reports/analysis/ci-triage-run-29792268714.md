### DS-1 — TC-007: Whitespace-only name

### DS-1 — TC-016: Rapid double-click Create

| Field | Detail |
|---|---|
| **Affected location** | `tests/ds1-create-program.spec.ts:292-317`; deployed Programs component `function O9`, create handler `z` in `/assets/index--ha_Sndr.js` at byte ~617035; Create button at byte ~631491 |

### DS-2 — TC-010: Whitespace-only edit name

| Field | Detail |
|---|---|
| **Affected location** | `tests/ds2-edit-program.spec.ts:212`; `pages/programs.page.ts:55-56` |
| **Classification** | **Test issue — locator/assertion issue** |
| **Root cause** | `programs.exactText('   ')` is a page-wide `getByText` query. Browser/Playwright text normalization makes a whitespace-only query match many unrelated layout nodes, so it cannot prove that no whitespace-named row exists. |
| **Expected** | No row with a whitespace-only name. |

| **Evidence** | `data/80bdf8b2a8a8270f28d7a891d952117e772173b5.md`, `data/d2681b9d86c2f912f73c2e96210b7614e38f2699.md`, `data/6903626fb39c3a06b4dbb819521d91b29c6de8f0.md`, trace `data/7f3197903b19f1dd37a5c7bef080b94058e0b07f.zip`. |
